# Security Policy

## Supported Versions

Currently, only the `main` branch and the latest released version of V-Dock receive security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

Security is a top priority for us. If you discover a security vulnerability within V-Dock (such as unsafe shell execution, privilege escalation, or exposed local data), please do NOT report it by opening a public GitHub issue.

Instead, please send an email to the repository owner directly.

We will try our best to address the vulnerability, issue a patch, and give you credit for the responsible disclosure.

**Please include the following information in your report:**
- Type of issue (e.g., shell injection, memory leak, privilege escalation).
- Full paths to the affected source code file(s).
- Step-by-step instructions to reproduce the issue.
- Proof of Concept (PoC) or exploit code if available.

Thank you for helping keep V-Dock safe!
